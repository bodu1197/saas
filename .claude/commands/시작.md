# /시작 — 작업 시작 전 필수 점검

이 커맨드는 매 작업 세션 시작 시 호출한다. (또는 사용자가 호출하도록 요청한다.)
모든 단계를 순서대로 수행하고, 각 단계의 결과를 사용자에게 보고한다.

## 1. 헌법 확인

`CLAUDE.md`를 읽었다고 보고. (이미 읽었으면 "확인" 한 줄.)

## 2. 학습된 교훈 확인 ⭐ (가장 중요)

`docs/LESSONS.md`를 처음부터 끝까지 읽고, 다음을 보고한다:

- 총 교훈 수
- 카테고리별 분포
- 재발 카운터 3 이상인 교훈 ID 목록 (특별히 주의해야 함)
- 자동 검증 룰이 활성화된 교훈 ID 목록

## 3. 인프라 정보 확인

`docs/infrastructure.md`의 빠른 참조 표가 채워져 있는지 확인.
미채움 항목이 있으면 사용자에게 보고.

## 4. 슬래시 커맨드 인지

`.claude/commands/` 디렉토리의 파일 목록을 한 번 훑고, 어떤 커맨드가 있는지 보고.

## 5. 현재 작업 환경 확인

- `git status` — 현재 브랜치, 변경사항
- main/master 브랜치인지 확인 (맞으면 즉시 경고)

## 6. 검증 시스템 동작 확인

`bash scripts/check-lessons.sh` 실행하여 통과하는지 확인.
실패하면 위반 사항 보고 (자동으로 카운터 증가됨).

## 7. 보고 형식

```
## 작업 시작 점검 완료

- 헌법: ✅
- 학습 교훈: N개 (그 중 3회 이상 재발: M개)
- 인프라: ✅ / ❌ (미채움 항목 있음)
- 슬래시 커맨드: [/리뷰5, /시작, ...]
- 브랜치: <브랜치명>
- 검증: ✅ / ❌

## 특별히 주의할 점 (3회 이상 재발 교훈)
- L-NNNN: <제목> — <어떻게 회피할지>
...

준비 완료. 작업을 진행할 수 있습니다.
```
