# /교훈추가 — 새로운 실수를 학습하기

방금 발생한 실수(또는 사용자가 지적한 실수)를 LESSONS.md에 기록한다.

## 사용 시점

- 사용자가 "그건 잘못된 방식이야", "다음부터는 ~게 해" 같은 피드백을 줄 때
- `pnpm verify`가 실패해서 같은 종류의 오류를 반복 수정하고 있을 때
- AI 스스로 "아, 이거 아까도 이랬는데" 하는 패턴을 인지했을 때
- 사용자가 명시적으로 `/교훈추가` 를 호출했을 때

## 절차

### 1. 유사 교훈 확인 (필수)

먼저 `docs/LESSONS.md`를 검색하여 비슷한 패턴의 교훈이 이미 있는지 확인.

- **있으면** → 새로 추가하지 말고 `bash scripts/bump-lesson.sh L-NNNN`으로 카운터만 +1.
- **없으면** → 2번으로 진행.

### 2. 교훈 작성 정보 수집

사용자에게 4가지를 자연어로 묻거나, 맥락에서 추출:

- **상황**: 어떤 작업 중에 발생했나
- **잘못된 행동**: 무엇을 잘못했나 (구체적 코드/명령어)
- **올바른 행동**: 어떻게 해야 했나
- **탐지 방법**: 다음에 어떻게 잡을 수 있나

카테고리 후보:

- `infra` — Supabase/Vercel/배포
- `type-safety` — TypeScript 타입
- `git` — 브랜치, 커밋, 워크트리
- `security` — 키, 인증, RLS
- `architecture` — 폴더 구조, 계층
- `api` — 라우트, 응답 형식
- `performance` — 빌드, 번들, 쿼리
- `ux` — UI 동작, 접근성
- `other`

심각도: `low` (스타일 수준) / `medium` (기능 영향) / `high` (보안·데이터) / `critical` (프로덕션 다운).

### 3. 스크립트로 추가

```bash
bash scripts/add-lesson.sh \
  "교훈 제목 (15자 이내)" \
  "카테고리" \
  "심각도" \
  "상황" \
  "잘못된 행동" \
  "올바른 행동" \
  "탐지 방법"
```

### 4. 자동화 가능 여부 즉시 판단

추가된 교훈을 보고 "이거 grep이나 ESLint로 잡을 수 있나?"를 즉시 판단.
가능하면 그 자리에서 `scripts/check-lessons.sh`에 패턴을 추가하고, LESSONS.md의 `auto_check: true`로 갱신.

### 5. 사용자에게 보고

```
## 교훈 학습됨

ID: L-NNNN
제목: ...
카테고리: ... (이 카테고리 누적: M개)
재발 시 자동 차단: ✅ 활성 (scripts/check-lessons.sh) / ⏳ 수동 모니터링

다음부터 같은 실수가 발생하면 verify 단계에서 즉시 차단됩니다.
```
