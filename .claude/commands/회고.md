# /회고 — 작업 종료 시 자동 회고

작업 한 단위(기능 구현, 버그 수정 등)를 마쳤을 때 호출한다.
이번 세션에서 학습할 만한 점이 있었는지 스스로 점검하고 LESSONS.md를 갱신한다.

## 절차

### 1. 이번 세션 활동 요약

방금 한 작업을 다음 관점에서 스스로 회고:

- `pnpm verify`가 처음에 실패한 적이 있었나? 이유는?
- 사용자가 도중에 "그게 아니야", "다시 해" 같은 피드백을 줬나?
- 같은 파일을 두 번 이상 수정한 적이 있었나? (= 처음에 잘못 만든 것)
- 의존성을 추가했다가 다시 제거한 적이 있나?
- 환경 변수, 인프라 정보를 묻고 싶은 충동이 있었나?

### 2. 학습할 점 도출

위 회고에서 "다음에 더 잘할 수 있는 것"이 1개 이상 있다면:

- 기존 LESSONS.md에 같은 패턴이 있는지 확인
- 있으면 `bump-lesson.sh`로 카운터 +1
- 없으면 `/교훈추가` 절차로 새 교훈 등록

### 3. 자동화 가능성 점검

방금 추가/갱신한 교훈을 다음 순서로 자동화 시도:

1. **타입 시스템으로 막을 수 있나?** → 타입 정의 추가
2. **ESLint로 막을 수 있나?** → `eslint.config.mjs`에 룰 추가
3. **grep으로 잡을 수 있나?** → `scripts/check-lessons.sh`에 패턴 추가
4. **위 셋 다 안 되나?** → 사람의 리뷰만 가능 (LESSONS.md에 명시)

### 4. 메타 패턴 검사

한 카테고리에 5개 이상 교훈이 누적되었으면 메타 패턴 기록.
예: "이 프로젝트의 AI는 `infra` 카테고리에서 자주 실수한다 → docs/infrastructure.md를 더 강하게 만들어야 함"

### 5. 보고 형식

```
## 작업 회고

이번 세션에서 학습한 점: N건
- 신규 교훈: L-NNNN ...
- 재발 카운터 증가: L-NNNN (이제 M회) ...

자동 검증 룰 추가됨: 0건 / N건

다음 세션 시작 시 /시작 커맨드로 LESSONS.md를 다시 읽으면, 위 교훈들이 적용됩니다.
```
